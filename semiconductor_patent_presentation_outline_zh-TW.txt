第 1 頁｜執行摘要：2021–2025 半導體專利競爭已從單點製程，轉向「封裝 × 系統 × 製造整合」
標題
- 2021–2025 全球廣義半導體專利：技術重心轉向先進整合

關鍵訊息
- 最穩定領先群：TSMC、Intel、Samsung
- 第二梯隊依主題分化：Micron、ASM、Lam Research、Applied Materials、Adeia、Amkor、ASML / Carl Zeiss SMT
- 高成長關鍵詞：chiplet、hybrid bonding、backside power、HBM、EUV

建議圖表
- 左：五大前沿主題成長倍率條圖
- 右：產業鏈競爭地圖（設計/封裝/製造/設備/光學）

講者重點
- 這一輪不是單一節點微縮競賽，而是系統整合能力競賽
- 專利布局已明顯往 3D 整合、先進封裝、記憶體頻寬、製程設備協同移動

管理實務意涵
- 技術管理單位不能再把 roadmap 只切成前段製程 / 後段封裝兩條線；要改成跨 BU 的整合型技術組合管理
- 專利評估要從「單一元件數量」改成「跨平台組合可防禦性」

第 2 頁｜競爭格局：誰在主導 2021–2025 半導體專利布局
標題
- 產業鏈雙核心：器件/封裝/IC 與 製程/設備 同步升級

已知事實
- Core 鏡像相對領先群：Intel、TSMC、Samsung、Adeia、Micron、Apple、Amkor
- Fab 鏡像相對領先群：TSMC、ASM、Lam Research、SanDisk/WD、Applied Materials
- TSMC 是少數在兩個鏡像都強的公司

建議圖表
- 兩欄排行：Core top assignees vs Fab top assignees
- 中間以連線標出橫跨兩側的公司（如 TSMC）

深入分析
- Intel 在 chiplet / 封裝整合存在感高，說明先進封裝不再只是 OSAT 主題，而是系統架構主題
- Samsung 與 Micron 的出現，反映記憶體與封裝耦合升高
- ASM / Lam / Applied Materials 靠前，表示設備供應商不只賣機台，也在鎖定製程 know-how 與未來議價權

管理實務意涵
- 研發管理上要把競爭者分成「產品競爭者」與「製造能力競爭者」兩張圖，不可混為一談
- 對供應鏈管理來說，設備商專利強度升高代表聯合開發時的 IP 邊界要更早談
- 企業若只追產品 patent count，會低估設備/材料商對未來節點節奏的主導力

第 3 頁｜技術熱點 1：先進封裝、chiplet、hybrid bonding 為成長主軸
標題
- 封裝已從成本中心變成架構創新中心

已知事實
- chiplet：1410 → 8964
- hybrid bonding：2902 → 8839
- 相關領先申請人包括 Intel、TSMC、Micron、AMD、Adeia、YMTC

建議圖表
- chiplet 與 hybrid bonding 雙折線
- 右側加一個 ecosystem stack：logic / memory / IO / advanced packaging

深入分析
- chiplet 的高成長代表效能提升路徑由單晶片擴張為多裸晶異質整合
- hybrid bonding 熱度升高，表示產業正往更細 pitch、更高頻寬、更低延遲的互連架構推進
- Adeia 這類鍵合技術公司能進前列，顯示關鍵價值正從單一晶片設計轉向互連與結合方式

管理實務意涵
- 研發組織應建立「封裝架構 PM / 系統封裝架構師」角色，而不是只靠製程或封裝工程師各自優化
- 技術投資評估要把 KGD、對位、rework、封裝內測試列為同級投資項，而不是配角
- 若公司做晶片產品，下一輪技術平台管理應從 die roadmap 升級為 package platform roadmap

第 4 頁｜技術熱點 2：HBM 與記憶體-運算耦合快速升高
標題
- HBM 不只是記憶體議題，而是 AI 系統瓶頸的專利映射

已知事實
- HBM：10025 → 27205
- 代表領先群：Samsung、Intel、Micron
- Core CPC 中 G11C（記憶體）靠前

建議圖表
- HBM 年度成長面積圖
- 右邊做「compute–memory bottleneck」示意圖

深入分析
- HBM 專利快速上升，與 AI 訓練/推論對高頻寬低延遲資料搬移的需求高度一致
- 記憶體議題已不再只屬於 memory vendor，而是牽動 GPU、封裝、中介層、散熱、供電的系統性主題
- 這使記憶體專利的重要性從零組件層提升到平台決勝層

管理實務意涵
- 技術管理必須把 memory architecture 納入平台級 KPI，而不是只看 compute die 指標
- 產品規劃上，應建立 compute / memory / package 聯合評審機制，避免局部最佳化
- 商務與策略上，可提早鎖定 HBM、interposer、散熱相關合作與授權談判窗口

第 5 頁｜技術熱點 3：EUV、沉積、光刻仍是先進製程底盤
標題
- 即使封裝升溫，製程設備鏈仍是不可替代的底層戰場

已知事實
- EUV：23146 → 30707
- Fab CPC 熱點：C23C、G03F、H01J、B82Y
- 代表領先群：TSMC、ASM、Lam Research、Applied Materials、ASML / Carl Zeiss SMT

建議圖表
- EUV 年度折線 + 製程鏈價值分布圖（曝光、沉積、蝕刻、量測）

深入分析
- EUV 高基數下仍持續成長，說明先進節點依然高度依賴曝光與周邊材料/光學/量測能力
- C23C、G03F 靠前，表示競爭不只在光刻設備本身，也在沉積、圖案轉移與製程整合
- Carl Zeiss SMT 與 ASML 的可見度，提醒「光學能力」也是晶片競爭力的一部分

管理實務意涵
- 技術發展管理不可過度偏向設計端；先進製程可得性本身就是產品策略邊界
- 若企業依賴先進製程代工，需建立節點風險雷達：曝光、材料、設備供應鏈、地緣風險缺一不可
- 與設備商/材料商的共同開發，不應只由製造單位主導，需納入法務、IP、產品平台單位共同治理

第 6 頁｜白區與新機會：未來 2–4 年最值得追的技術空間
標題
- 真正的白區在「量產化輔助技術」與「跨層整合治理」

五個白區
- Chiplet 系統層協同：互連協議、跨 die 時序/電源/熱管理、封裝內測試、容錯
- Hybrid bonding 量產化輔助：對位、瑕疵檢測、yield learning、rework、KGD 流程
- Backside power 的 DFT / reliability / repair
- 先進封裝中的材料與中介層方案：低翹曲、熱界面、molded interposer、局部修補
- SiC / GaN 功率器件的模組化與可靠性

建議圖表
- 2x3 白區地圖：技術成熟度 vs 競爭擁擠度

深入分析
- 最有機會的不是已被巨頭高度壟斷的結構本體，而是把技術推到量產可控所需的輔助能力
- 也就是說，下一輪價值會從「發明一個新結構」部分轉向「讓它可大量製造、可測、可修、可保證可靠度」

管理實務意涵
- 技術投資組合要預留 20–30% 給 manufacturing-enablement 類專案
- IP 團隊應主動搜尋 reliability、DFT、rework、yield-learning 類專利空隙，而非只盯主結構 claim
- 新事業探索可從大廠主航道旁的輔助技術切入，降低正面硬碰風險

第 7 頁｜對管理層的實務建議：如何把專利洞察轉成技術發展機制
標題
- 從專利情報到技術治理：四個管理動作

管理動作
1. 重構技術路線圖
- 把 roadmap 改成 device / package / process / system 四層連動，而非單線研發計畫

2. 建立跨部門專利雷達
- 每季固定追蹤 chiplet、HBM、hybrid bonding、backside power、EUV 五大主題
- 將競爭者分成：產品公司、設備公司、材料/光學公司、研究型 IP 公司

3. 調整研發投資組合
- 核心結構創新
- 量產化輔助技術
- 測試/可靠性/可修復性
- 供應鏈共同開發
四類都要有，不可只押第一類

4. 專利與產品開發同步治理
- 在 NPI / NPD gate 中加入 FTO、hotspot check、white-space review、co-development IP review

建議圖表
- 四象限管理框架圖：技術、產品、製造、IP 治理

管理實務意涵
- 研發管理成熟度的關鍵，不再只是研發效率，而是跨層決策速度
- 能把專利情報接進 roadmap 與投資組合管理的公司，會更早避開紅海、鎖定白區、提高開發命中率

第 8 頁｜結論與決策建議
標題
- 給管理層的三個決策：擴整合、補量產、做平台

結論
- 2021–2025 半導體專利競爭主軸已明顯移向先進整合
- 真正的勝負手不只是元件創新，而是封裝、記憶體、製程、設備、光學、可靠性協同
- 管理上最值得加碼的是：跨層 roadmap、量產化輔助技術、平台式合作與 IP 治理

決策建議
- 短期（0–12 個月）：建立五大主題專利雷達與競爭者地圖
- 中期（12–24 個月）：把 white-space 轉為研發立項池與合作名單
- 長期（24+ 個月）：把技術平台治理從單 die 思維升級為 system/package/process 共治
